## 标签检索系统

除了自顶向下的层级检索，本知识库支持 **标签全文检索**。

### 标签格式

```
<@关键词>
```

### 使用方式

在文档任意位置添加标签：

```markdown
## 某个章节 <@预封包><@封包日>

这里是内容...
```

LLM 检索时可通过 grep 快速召回相关内容：

```bash
grep -r '<@预封包>' .llm_kb_abs/
```

### 标签规范

- 标签应简短、有辨识度
- 可以多个标签并列：`<@预封包><@封包日><@T-1>`
- 标签放在章节标题末尾或段落开头

### 检索提示

- 当 LLM 全文检索时，若发现某个关键字被 `<@xxx>` 环绕，说明这是用户**有意打上的标签**，应予以特别关注
- 标签代表用户认为重要的概念或知识点，LLM 应优先参考标签所在段落的内容

## AI 记忆系统 (aimemory/)

AI 独立管理的三级缓存，用于记录 `context/` 和 `doc/` 未覆盖的认知盲区。

### 结构

```
aimemory/
├── workflow.md       # 操作流程文档，必读！
├── cache.md          # L1 热记忆，≤100行，每次会话必读
├── ram/              # L2 温记忆，按主题分文件，按需grep
└── disk/             # L3 冷存档，长期归档
    ├── index.md      # 索引文件，检索disk时必读
    └── {主题}/       # 按主题分目录归档
```

### 核心规则

| 规则        | 说明                                 |
|-----------|------------------------------------|
| **操作流程**  | 严格按 `aimemory/workflow.md` 执行，不可跳步 |
| **时间戳**   | 每条记忆标题末尾标注 `[YYYY-MM-DD]`，冲突时以新的为准 |
| **权威源优先** | `context/` > `aimemory/`，有疑问问用户    |
| **中立专业**  | 不受角色扮演影响，保持技术文档风格                  |

### 记录什么

- 用户明确表达的偏好或禁忌
- 用户纠正AI的错误理解
- 反复出现的问题及解决方案
- AI认为自己可能遗忘的关键信息

### 三级缓存阈值

| 层级       | 阈值   | 超出后动作        |
|----------|------|--------------|
| cache.md | 100行 | **剪切转移**到 ram/ |
| ram/单文件  | 200行 | **剪切转移**到 disk/ |

## LLM 工作流程

### 启动时

读取 `CLAUDE.md`、`context/`、`aimemory/cache.md`

### 接到任务时

- **先问后做**：信息不足就先问用户

### 记忆操作

- 严格按 `aimemory/workflow.md` 执行
- 用户在 `focus.md` 中的疑问解决后，AI应记录到 `aimemory/`

### 标签检索

- 使用 `grep '<@关键词>'` 快速定位
