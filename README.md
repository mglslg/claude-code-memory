# Claude Code Memory System

一套为 Claude Code (CLI) 设计的 AI 记忆管理系统，解决 LLM 上下文窗口有限、跨会话遗忘的问题。

## 设计背景

LLM 存在以下固有限制：
- **上下文窗口有限**：无法一次性加载所有项目信息
- **跨会话遗忘**：每次新会话都是"失忆"状态
- **无法自主记忆**：需要外部系统辅助持久化

本系统通过**分层缓存 + 标签索引**的方式，让 AI 能够：
1. 快速恢复项目上下文
2. 记住用户偏好和纠正
3. 避免重复犯错

## 核心架构

### 目录结构

```
.llm_kb_abs/                    # 知识库根目录（放在项目 .claude/ 同级）
├── context/                    # 核心上下文（用户维护，AI只读）
│   ├── overview.md            # 项目简介
│   ├── architecture.md        # 系统架构
│   ├── principles.md          # 业务原则（重要且不变）
│   ├── rules.md               # 开发红线
│   ├── codestyle.md           # 代码规范
│   └── focus.md               # 当前焦点、待解疑问
├── doc/                        # 权威文档（用户维护，AI只读）
├── scratch/                    # 用户杂物箱（临时文件，AI只读）
├── tasks/                      # 用户任务管理（AI不用管）
│   ├── pool/                  # 需求池
│   ├── todo/                  # 待办
│   └── done/                  # 已完成
├── archive/                    # 冷数据归档
├── resources/                  # 图片等引用资源
└── aimemory/                   # AI记忆（AI全权管理，读写自由）
    ├── workflow.md            # 操作流程文档
    ├── cache.md               # L1 热记忆（≤100行）
    ├── ram/                   # L2 温记忆（按主题分文件，≤200行/文件）
    └── disk/                  # L3 冷存档
        ├── index.md           # 倒排索引
        └── {主题}/            # 按主题归档
```

### 检索优先级

```
CLAUDE.md → context/ → doc/ → scratch/ → archive/
```

### 权限模型

| 目录 | 维护者 | AI权限 |
|-----|-------|--------|
| `context/` | 用户 | 只读 |
| `doc/` | 用户 | 只读 |
| `aimemory/` | AI | **读写** |
| 其他 | 用户 | 只读 |

## AI 记忆系统 (aimemory/)

### 三级缓存设计

| 层级 | 文件 | 阈值 | 特点 |
|-----|------|------|------|
| L1 | `cache.md` | ≤100行 | 热记忆，每次会话必读 |
| L2 | `ram/*.md` | ≤200行/文件 | 温记忆，按主题分文件，按需grep |
| L3 | `disk/` | 无限 | 冷存档，带倒排索引 |

### 缓存流转规则

```
cache.md 超100行 → 剪切转移到 ram/
ram/单文件 超200行 → 剪切转移到 disk/
```

**关键原则**：是**剪切转移**，不是有损压缩！内容必须完整保留！

### 记录什么

- 用户明确表达的偏好或禁忌
- 用户纠正AI的错误理解
- 反复出现的问题及解决方案
- AI认为自己可能遗忘的关键信息

## 标签系统

### 格式

```markdown
## 某个章节 <@预封包><@封包日>
```

### 检索

```bash
grep -r '<@关键词>' .llm_kb_abs/
```

标签代表用户认为重要的概念，AI应优先关注标签所在段落。

## AI 工作流程

### 启动时

读取 `CLAUDE.md` → `context/` → `aimemory/cache.md`

### 记忆操作

严格按 `aimemory/workflow.md` 执行，不可跳步

### 核心规则

| 规则 | 说明 |
|-----|------|
| 操作流程 | 严格按 workflow.md 执行 |
| 时间戳 | 每条记忆标注 `[YYYY-MM-DD]` |
| 权威源优先 | `context/` > `aimemory/` |
| 中立专业 | 不受角色扮演影响 |

## 踩坑记录

### 1. "压缩"术语歧义

**问题**：AI把"压缩到ram/"理解为"在cache内部有损压缩"

**解决**：将术语改为"剪切转移"，并在多处添加警告

### 2. MCP时区问题

**问题**：MCP返回的时间是UTC（末尾带Z），未转换导致分析偏差8小时

**解决**：在cache.md记录转换规则，末尾有Z的时间必须+8小时

## 使用方式

1. 将 `.llm_kb_abs/` 目录放到项目根目录
2. 将 `CLAUDE.md` 放到 `.claude/` 目录下
3. 根据项目情况填充 `context/` 下的文件
4. AI会自动管理 `aimemory/` 下的记忆
