# AI Memory Workflow

本文档定义了AI记忆系统的完整操作流程，**必须严格按步骤执行，不可跳步**。

---

## 一、记忆检索流程

### 触发条件
- 需要查找历史信息但cache.md中没有
- 用户主动说"查一下记忆"、"之前有没有记录"等

### 执行步骤

#### Step 1: 检查 cache.md
```
grep "关键词" cache.md
```
- [ ] 若命中 → 返回结果，**流程结束**
- [ ] 若未命中 → 进入 Step 2

#### Step 2: 检查 ram/
```
grep -r "关键词" ram/
```
- [ ] 若命中 → 返回结果，**流程结束**
- [ ] 若未命中 → 进入 Step 3

#### Step 3: 读取 disk/index.md
- [ ] 查找标签索引或主题索引，定位目标文件
- [ ] 若无匹配 → 返回"未找到相关记忆"，**流程结束**
- [ ] 若有匹配 → 记录目标文件路径，进入 Step 4

#### Step 4: 读取目标文件
- [ ] 先读文件头（前15行）确认相关性
- [ ] 若相关 → 读取所需内容，返回结果
- [ ] 若不相关 → 返回 Step 3 检查下一个匹配项

## 二、记忆写入流程

### 触发条件
- 发现值得记住的信息（用户偏好、纠正、重要发现等）
- 用户对你的失忆表现出不耐烦时
- 用户对你的项目理解表现出不耐烦时
- 用户主动要求你记录在记忆里，或者让你写在小本本上

### 执行步骤

#### Step 1: 写入 cache.md
- [ ] 在cache.md末尾追加新条目
- [ ] 格式：`## 标题 <@标签> [YYYY-MM-DD]`
- [ ] 检查cache.md行数，若≤100行 → **流程结束**
- [ ] 若>100行 → 进入 Step 2

#### Step 2: 归档转移 cache → ram（⚠️ 是剪切，不是删减！）
- [ ] 识别可归档内容（非高频、已解决、参考类）
- [ ] 确定目标分类文件（如 `ram/api_reference.md`）
- [ ] 若分类文件不存在 → 创建新文件
- [ ] 检查目标ram文件行数，若≤200行 → 进入 Step 3
- [ ] 若>200行 → 先执行 Step 4，再回到 Step 3

#### Step 3: 执行转移
- [ ] **剪切**内容从cache.md到目标ram文件（内容完整保留，不可删减！）
- [ ] 保留时间戳和标签
- [ ] ⚠️ **禁止在cache.md内部做有损压缩！不许删内容来凑行数！**
- [ ] **流程结束**

#### Step 4: 归档 ram → disk
- [ ] 读取 disk/index.md（若不存在则创建）
- [ ] 确定归档路径：`disk/{主题}/archive_YYYY-MM.md`
- [ ] 若目录不存在 → 创建目录
- [ ] 将ram文件内容追加到归档文件
- [ ] 更新 disk/index.md 索引
- [ ] 清空该ram文件（保留文件头注释）
- [ ] 返回 Step 3

## 三、disk/index.md 结构

```markdown
# Disk Index

> 最后更新：YYYY-MM-DD

## 按标签索引

| 标签 | 文件位置 |
|-----|---------|
| <@OSS> | env_config/archive_2025-02.md |
| <@灰度> | api_reference/archive_2025-02.md |

## 按主题索引

| 主题 | 最新归档 | 条目数 |
|-----|---------|-------|
| api_reference | 2025-02 | 5 |
| pitfalls | 2025-02 | 3 |
```

---

## 四、归档文件结构

每个归档文件开头必须有元数据头：

```markdown
# {主题} Archive - YYYY-MM

> **包含内容**：简要描述本文件包含的知识点
> **标签**：<@标签1><@标签2>
> **条目数**：N

---

## 具体条目...
```

---

## 五、注意事项

1. **时间戳必须标注**：每条记忆标题末尾 `[YYYY-MM-DD]`
2. **标签要打**：便于索引检索
3. **中立专业**：不受角色扮演影响，保持技术文档风格
4. **冲突处理**：`context/` > `aimemory/`，有疑问问用户
